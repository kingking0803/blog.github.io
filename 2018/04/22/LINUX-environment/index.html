LINUX environment

注意：如果不是root账号需要用sudo命令执行

安装jdk

1、检查版本

java -version

2、创建jdk存放位置

mkdir /usr/lib/jvm

3、下载jdk安装包到指定文件夹下，进入该文件夹，解压文件到指定文件夹下

tar -zxvf jdk-8u171-linux-x64.tar.gz -C /usr/lib/jvm

4、设置环境变量

vim /etc/profile

在最前面加上如下代码：

export JAVA_HOME=/usr/lib/jvm/jdk1.8.0_171
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:\${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=\${JAVA_HOME}/bin:$PATH

5、执行profile文件，即可以使得配置文件立即生效，不用重启电脑

source /etc/profile

6、检测jdk安装包

rpm -qa | grep java

7、卸载jdk

rpm -e –nodeps 安装包的名字

或者

yum remove 安装包的名字（这里可以用通配符匹配）

安装mysql

1、下载mysql安装包，并解压到/usr/local/目录下

tar -zxvf mysql-8.0.11-linux-glibc2.12-x86_64.tar.gz -C /usr/local/

2、修改文件名字

mv mysql-8.0.11-linux-glibc2.12-x86_64 mysql

3、检查mysql组和用户，如果不存在则创建

cat /etc/group | grep mysql
cat /etc/passwd | grep mysql

创建

groupadd mysql
useradd -r -g mysql mysql

设置权限

cd /usr/local/
chown -R mysql mysql/
chgrp -R mysql mysql/

4、在MySQL目录下添加data目录

mkdir data

5、更改mysql目录下所有文件的所属组和用户

chown -R mysql mysql/
chgrp -R mysql mysql/

6、安装和初始化数据库

./mysqld –initialize –user=mysql –basedir=/usr/local/mysql/ –datadir=/usr/local/mysql/data/

如果报错[ERROR] Can’t find error-message file ‘/usr/local/mysql/–datadir=/usr/local/mysql/data/share/errmsg.sys’.
则用如下命令再重试一遍

./mysqld –initialize –user=mysql –basedir=/usr/local/mysql/ –datadir=/usr/local/mysql/data/ –lc_message_dir=/usr/local/mysql/share –lc_message=en_US

这一步要记住产生的数据库密码，后面会用到，例如Cl;Y-fXrh3sB

7、启动服务

./mysqld_safe -u root -p
输入上一步产生的密码

8、修改密码

alter user ‘root‘@’localhost’ identified by ‘newpassword’;

9、将mysqld服务加入开机自启动项
将{mysql}/ support-files/mysql.server 拷贝为/etc/init.d/mysql并设置运行权限，这样就可以使用service mysql命令启动/停止服务，
否则就只能使用{mysql}/bin/mysqld_safe &命令来启动服务
还需要把mysql.server中basedir的相关路径，改为自定义的路径，默认路径是/usr/local/mysql

cp mysql.server /etc/init.d/mysql
chmod +x /etc/init.d/mysql
chkconfig –add mysql

10、启动服务

service mysql start

搭建web项目

1、下载tomcat，将其拷贝到home文件夹下，启动tomcat

./catalina.sh run

2、停止tomcat

./catalina.sh stop

还有其它的启动方式，这种启动方式可以在控制台输出日志

3、如果想在本机访问虚拟机的项目，可以配置网络未NET，然后访问虚拟机的IP地址，如果还是访问不了可能是虚拟机防火墙限制了该端口，可用如下命令开启

firewall-cmd –zone=public –add-port=8081/tcp –permanent

然后重启防火墙

firewall-cmd –reload
